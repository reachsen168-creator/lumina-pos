
Build a simple POS web app with Node.js + SQLite.

Requirements:
--

:::writing{variant="standard" id="10458"}
POS + INVENTORY + DELIVERY MANAGEMENT SYSTEM
FINAL COMPLETE REQUIREMENT

--------------------------------------------------

SYSTEM PURPOSE

This system manages product inventory, sales invoices, customer purchase history, delivery trips, damaged products, invoice item transfers, and reports.

The system must be simple, fast, mobile-friendly, and suitable for daily business use.

All records must support search, edit, delete, and history tracking.

Make the UI mobile friendly
Use vertical layout
Large touch buttons

--------------------------------------------------

MAIN SYSTEM MODULES

Dashboard
Products
Categories
Sales / Invoices
Customer Purchase History
Delivery Management
Damaged Items
Invoice Item Transfers
Reports
History Logs
Backup & Restore

--------------------------------------------------

PRODUCT MANAGEMENT

Each product must include:

id
name
category_id
base_price
track_stock (true / false)
stock_qty
created_date

Categories must include:

id
name

Products page must support:

Search box
Category filter
Date filter

Actions:

Add product
Edit product
Delete product

Deleting must require confirmation popup.

Invoice item fields:

id

qty

subtotal

Stock behavior:

If track_stock = true

--------------------------------------------------

AUTO LAST SALE PRICE

When selecting a product for a customer:

If the same customer previously bought the same product,
the system must automatically use the last sale price.

If no previous sale exists,
use the base price.

The system must also show:

Last sale to customer
Price
Date

User can still manually edit the price.

--------------------------------------------------

QUICK PRODUCT HISTORY FROM SALE PAGE

Inside the sale page users must be able to quickly view purchase history of the selected product for that customer.

Example:

Customer: Reach

2026-03-01
Beer = 1 x 3$

2026-03-05
Beer = 2 x 3.5$

--------------------------------------------------

DAMAGED ITEMS MANAGEMENT

The system must allow recording damaged items.

Fields:

id
product_id
qty
invoice_id
customer_name
damage_reason
date
status

Status:

Damaged
Repaired
Sold Again

Damaged items must be separated from normal stock.

If a damaged item is sold again, system must display warning:

"This item was previously damaged."

Invoice must include note that the item was previously damaged.

--------------------------------------------------

INVOICE ITEM TRANSFER

Users must be able to transfer items from one invoice to another.

Example:

Original invoice:

INV-001
Reach

Beer = 10

Transfer 4 Beer to another customer.

Result:

INV-001

Beer = 6

New invoice:

INV-020
Dara

Beer = 4

IMPORTANT RULE:

Stock must NOT change during invoice transfer.
